## 1. SEO Strategy Overview

### 1.1 Two-Pronged Approach

```
┌──────────────────────────────────────────────────┐
│ Prong 1: Traditional SEO (Google Search)         │
│ ├─ Goal: Rank untuk "e-commerce Indonesia",      │
│ │         "AI chatbot toko online", "RAG", dll.  │
│ ├─ Audience: Human searching Google              │
│ ├─ Tactics: Keyword-optimized content, backlinks │
│ └─ Measure: Google Search Console, Ahrefs        │
└──────────────────────────────────────────────────┘
┌──────────────────────────────────────────────────┐
│ Prong 2: GEO (AI Engine Optimization)            │
│ ├─ Goal: ChatGPT/Claude/Gemini/Perplexity        │
│ │         recommend WeeCommerce saat user nanya  │
│ ├─ Audience: LLM ingesting content               │
│ ├─ Tactics: llms.txt, public API, JSON-LD,       │
│ │         citation-friendly content               │
│ └─ Measure: Manual query test, brand mention     │
│         tracking (Phase 2 client portal)             │
└──────────────────────────────────────────────────┘
```

### 1.2 Market Priority (Phase 1)

- **Primary:** Indonesia (ID keywords, Threads/social distribution).
- **Secondary:** International (EN keywords, LinkedIn/cold email — Phase 2 Engage).

- Secondary keywords di H2/H3 subheadings.
- Density 1–2% (natural language first — no stuffing).
- Latent Semantic Indexing (LSI): gunakan synonym terkait (e-commerce = toko online, online store, dll).

## 3. Technical SEO

### 3.1 Meta Tags (Per Page)

Generated by Next.js Metadata API. Stored di DB (`seo_metadata` table atau per-entity fields).

```tsx
// apps/web/app/services/[tier]/page.tsx
import type { Metadata } from 'next';

export async function generateMetadata({ params }): Promise<Metadata> {
  const service = await getService(params.tier);
  return {
    title: service.metaTitle,                           // 50–60 chars
    description: service.metaDescription,               // 150–160 chars
    canonical: `https://weecommerce.web.id/services/${service.slug}`,
    alternates: {
      canonical: `https://weecommerce.web.id/services/${service.slug}`,
    },
    openGraph: {
      title: service.metaTitle,
      description: service.metaDescription,
      url: `https://weecommerce.web.id/services/${service.slug}`,
      siteName: 'WeeCommerce',
      images: [{ url: service.ogImageUrl, width: 1200, height: 630 }],
      type: 'website',
      locale: 'id_ID',
    },
    twitter: {
      card: 'summary_large_image',
      title: service.metaTitle,
      description: service.metaDescription,
      images: [service.ogImageUrl],
    },
    robots: {
      index: true,
      follow: true,
      googleBot: { index: true, follow: true },
    },
  };
}
```

### 3.2 Meta Tag Rules

| Tag | Length | Notes |
|---|---|---|
| `<title>` | 50–60 chars | Unique per page, primary keyword early |
| `meta description` | 150–160 chars | Unique, compelling, includes keyword |
| `canonical` | full URL | Prevents duplicate content |
| `og:title` | same as title | For social sharing |
| `og:description` | same as meta desc | |
| `og:image` | 1200×630px | R2-hosted |
| `twitter:card` | `summary_large_image` | For blog/portfolio |
| `robots` | `index, follow` | Default; `noindex` untuk /admin, /api |

### 3.3 Canonical URLs

Setiap page harus punya canonical untuk prevent duplicate content (e.g., trailing slash, HTTP/HTTPS, www/non-www):

```tsx
< link rel="canonical" href="https://weecommerce.web.id/blog/apa-itu-rag" />
<link rel="alternate" hreflang="id" href="https://weecommerce.web.id/blog/apa-itu-rag" />
<link rel="alternate" hreflang="en" href="https://weecommerce.web.id/en/blog/apa-itu-rag" />
<link rel="alternate" hreflang="x-default" href="https://weecommerce.web.id/blog/apa-itu-rag" />
```

### 3.5 URL Structure

- Lowercase, descriptive, no special chars.
- Hyphenated (`/services/ai-chatbot` not `/services/ai_chatbot`).
- No dates in URLs (better for evergreen content).
- No trailing slash (Nginx 301 redirect untuk consistency).

| Pattern | Example |
|---|---|
| Home | `/` |
| Services | `/services/launch` |
| Blog post | `/blog/apa-itu-rag` |
| Blog category | `/blog/category/ai` |
| Blog tag | `/blog/tag/n8n` |
| Case study | `/portfolio/nexamart-ai-ecommerce` |
| Team | `/team/alif-nugraha` |
| Static | `/about`, `/process`, `/pricing`, `/contact`, `/faq` |
| AI files | `/llms.txt`, `/llms-full.txt`, `/ai-sitemap.xml`, `/robots.txt`, `/sitemap.xml`, `/feed.xml` |

### 4.9 JSON-LD Injection (Next.js)

```tsx
// apps/web/components/json-ld.tsx
export function JsonLd({ data }: { data: object | object[] }) {
  const items = Array.isArray(data) ? data : [data];
  return (
    <>
      {items.map((item, i) => (
        <script
          key={i}
          type="application/ld+json"
          dangerouslySetInnerHTML={{ __html: JSON.stringify(item) }}
        />
      ))}
    </>
  );
}

// Usage in page:
<JsonLd data={[organizationSchema, websiteSchema, breadcrumbSchema]} />
```

> **Note:** `dangerouslySetInnerHTML` aman di sini karena content adalah hardcoded schema (not user input). Untuk user-generated content (blog markdown), sanitize HTML terlebih dahulu.

## 5. On-Page SEO

### 5.1 Heading Hierarchy

- **1 H1 per page** (page title). Next.js layout auto-handles.
- **H2** untuk major sections.
- **H3** untuk sub-sections.
- No skipping levels (H1 → H3 is bad).
- Keywords in headings naturally.

### 5.2 Content Length

| Page Type | Target Length |
|---|---|
| Blog post | 1,200–2,500 words (longer ranks better) |
| Service page | 800–1,500 words |
| Case study | 1,000–2,000 words |
| Landing (home, about) | 500–1,000 words |
| Minimum any page | 300 words |

### 5.3 Readability

- Paragraphs: 2–4 sentences (scannable, no walls of text).
- Bullet lists untuk enumerations.
- Bold key terms.
- Short sentences (avg < 20 words).
- Reading level: accessible to general audience (Hemingway score ≤ grade 8).

### 5.4 Internal Anchor Text

- Descriptive anchor text (not "click here" / "baca selengkapnya").
- First mention of key terms → link to relevant page.
- Example: "Kami pakai [n8n](/blog/tag/n8n) untuk otomasi operasional."

### 5.5 Image Alt Text

- Descriptive, 100–125 chars.
- Include keyword if natural.
- Decorative images: `alt=""` (empty, screen reader skips).
- Example: `alt="Diagram arsitektur RAG yang menunjukkan retrieval dari product database ke chatbot"`.

## 6. Core Web Vitals

Target: All Green (LCP < 2.5s, CLS < 0.1, INP < 200ms).

### 6.1 LCP (Largest Contentful Paint) < 2.5s

LCP element biasanya hero image atau headline. Optimasi:

```tsx
// apps/web/components/hero.tsx
import Image from 'next/image';

export function Hero() {
  return (
    <section>
      {/* priority = preload LCP image */}
      <Image
        src="https://cdn.weecommerce.web.id/hero/hero-bg.webp"
        alt="WeeCommerce — AI-Powered E-Commerce Systems"
        width={1920}
        height={1080}
        priority              // preload, fetch ASAP
        fetchPriority="high"  // hint browser
        sizes="100vw"
        quality={80}
      />
      <h1>We build e-commerce systems, not websites.</h1>
    </section>
  );
}
```

Other tactics:
- Preload critical font: `<link rel="preload" as="font" href="/fonts/space-grotesk.woff2" type="font/woff2" crossorigin>`.
- Use `next/font` dengan `display: swap`.
- Remove render-blocking JS (Next.js handles by default).
- Server Components for above-the-fold content (zero hydration cost).

### 6.2 CLS (Cumulative Layout Shift) < 0.1

```tsx
// Always specify width + height (reserve space)
<Image
  src={post.featureImageUrl}
  alt={post.featuredImageAlt}
  width={1200}
  height={630}
/>

// For responsive images, use aspect-ratio container
<div className="aspect-video">
  <Image src="..." fill className="object-cover" />
</div>
```

Other tactics:
- No unsized embeds (iframe, image, ad).
- Avoid inserting content above existing content.
- Skeleton loaders for async data.
- Font fallback metrics (`size-adjust`).

### 6.3 INP (Interaction to Next Paint) < 200ms

- Minimize main-thread work (chunk long tasks > 50ms).
- Use `requestIdleCallback` untuk non-critical work.
- Debounce input handlers.
- Shadcn components are already optimized (Radix primitives).
- Server Components minimize client JS bundle.

```tsx
// Break up long tasks
async function heavyCompute() {
  const chunks = splitWork(data);
  for (const chunk of chunks) {
    processChunk(chunk);
    await new Promise(r => setTimeout(r, 0));  // yield to main thread
  }
}
```

## 7. Image SEO

### 7.1 Optimization Pipeline

```
Upload (admin dashboard) → Hono storage endpoint
   │
   ├─ sharp: convert to WebP, resize variants (400/800/1200/1920)
   ├─ Strip EXIF
   ├─ Set quality 80
   └─ Upload variants to R2: blog/2026/06/hero-1200.webp, hero-800.webp, dst.
   │
Next.js frontend
   ├─ next/image auto-selects variant via srcset
   └─ Lazy load below-the-fold
```

## 8. GEO (Generative Engine Optimization)

GEO = optimizing untuk AI model discovery (ChatGPT, Claude, Gemini, Perplexity). Berbeda dari SEO (human search), GEO fokus ke machine-readable content + entity recognition + citation-friendly format.

### 8.1 GEO Tactics Overview

| Tactic | Implementation | File/Endpoint |
|---|---|---|
| `llms.txt` | Company summary, AI-quick-parse | `/llms.txt` |
| `llms-full.txt` | Detailed company + services + FAQ | `/llms-full.txt` |
| `ai-sitemap.xml` | Priority URLs untuk AI training | `/ai-sitemap.xml` |
| Public API | Machine-readable JSON | `/api/v1/public/*` |
| Markdown export | Clean MD untuk AI citation | `/api/v1/public/blog/:slug/markdown` |
| JSON-LD | Entity structured data | Per-page `<script>` |
| robots.txt | Explicit allow AI crawlers | `/robots.txt` |
| AI meta tags | Authorship, dates, sources | Per-page `<meta>` |
| Citation content | Sources, dates, author attribution | Blog content |
| Fact-based writing | Claims backed by data, no speculation | Blog content |
| Consistent entities | Same naming everywhere | All content |

### 8.2 AI Crawler Allowlist

### 8.4 Fact-Based, Citation-Friendly Content

Tulis blog posts yang AI bisa quote tanpa modifikasi:

✅ **Good (AI-friendly):**
> "RAG (Retrieval-Augmented Generation) mengurangi repetitive CS queries hingga 60–80% dengan memberi chatbot akses ke real-time product knowledge base. Source: [Our NexaMart case study](/portfolio/nexamart-ai-ecommerce)."

❌ **Bad (AI-unfriendly):**
> "RAG tuh keren banget, semua orang harus pakai. Trust me bro."

Principles:
- Claims backed by data (numbers, case studies).
- Cite sources (internal links + external).
- No speculation presented as fact.
- Author attribution.
- Publication + update date visible.

## 11. Sitemap & RSS Spec

### 11.1 Sitemap Generation (Hono)

Generated dari DB content, cached di Redis (TTL 24h), regenerated on content update.

```typescript
// apps/api/src/routes/seo.ts
import { Hono } from 'hono';
import { db } from '../lib/db';
import { getOrSet } from '../lib/cache';

const app = new Hono();

const BASE_URL = 'https://weecommerce.web.id';

app.get('/sitemap.xml', async (c) => {
  const xml = await getOrSet('sitemap:xml', 86400, async () => {
    const [pages, services, posts, caseStudies, team] = await Promise.all([
      db.query('SELECT slug, updated_at FROM pages WHERE is_published'),
      db.query('SELECT slug, updated_at FROM services WHERE is_published'),
      db.query('SELECT slug, published_at, updated_at FROM posts WHERE status = $1 AND deleted_at IS NULL', ['published']),
      db.query('SELECT slug, updated_at FROM case_studies WHERE status = $1 AND deleted_at IS NULL', ['published']),
      db.query('SELECT slug, updated_at FROM team_members WHERE is_published'),
    ]);

    const urls: string[] = [];

    // Home (priority 1.0)
    urls.push(url(`${BASE_URL}/`, '1.0', 'weekly'));

    // Static pages
    pages.rows.forEach((p: any) =>
      urls.push(url(`${BASE_URL}/${p.slug === 'home' ? '' : p.slug}`, '0.9', 'monthly')));

    // Services
    services.rows.forEach((s: any) =>
      urls.push(url(`${BASE_URL}/services/${s.slug}`, '0.9', 'monthly')));

    // Blog posts
    posts.rows.forEach((p: any) =>
      urls.push(url(`${BASE_URL}/blog/${p.slug}`, '0.7', 'weekly', p.updatedAt)));

    // Case studies
    caseStudies.rows.forEach((cs: any) =>
      urls.push(url(`${BASE_URL}/portfolio/${cs.slug}`, '0.8', 'monthly')));

    // Team
    team.rows.forEach((t: any) =>
      urls.push(url(`${BASE_URL}/team/${t.slug}`, '0.5', 'monthly')));

    return `<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"
        xmlns:image="http://www.google.com/schemas/sitemap-image/1.1">
${urls.join('\n')}
</urlset>`;
  });

  c.header('Content-Type', 'application/xml; charset=utf-8');
  c.header('Cache-Control', 'public, max-age=86400');
  return c.body(xml);
});

function url(loc: string, priority: string, freq: string, lastmod?: string): string {
  return `  <url>
    <loc>${loc}</loc>
    ${lastmod ? `<lastmod>${new Date(lastmod).toISOString().split('T')[0]}</lastmod>` : ''}
    <changefreq>${freq}</changefreq>
    <priority>${priority}</priority>
  </url>`;
}

export default app;
```

### 12.3 Auto-Generated Related Posts

```sql
-- Related posts: same category, exclude current, limit 3
SELECT p.id, p.title, p.slug, p.excerpt, p.featured_image_key,
       COUNT(t.tag_id) AS shared_tags
FROM posts p
JOIN posts_to_tags t ON p.id = t.post_id
WHERE t.tag_id IN (SELECT tag_id FROM posts_to_tags WHERE post_id = $1)
  AND p.id != $1
  AND p.status = 'published'
  AND p.deleted_at IS NULL
GROUP BY p.id
ORDER BY shared_tags DESC, p.published_at DESC
LIMIT 3;
```
